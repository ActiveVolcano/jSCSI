 /**
 33   
  * <h1>CRC32CDigest</h1>
 34   
  * <p>
 35   
  * This class implements a Cyclic Redundancy Code (CRC) with the following
 36   
  * generator polynom <code>0x11EDC6F41</code>.
 37   
  * <code>G(x) = x**32 + x**28 + x**27 + x**26 + x**25 + x**23 + x**22 + </code>
 38   
  * <code>x**20 + x**19 + x**18 + x**14 + x**13 + x**11 + x**10 + x**9 + </code>
 39   
  * <code>x**8 + x**6 + 1</code> This polynom has a degree of <code>32</code> and
 40   
  * is defined in the iSCSI Standard (RFC3720). Further, this class implements
 41   
  * the algorithm from Intel, which is called Slicing-by-4. This algorithm
 42   
  * increases the performance of the CRC calculation by a factor of
 43   
  * <code>2</code> by using <code>4KB</code> of memory for the look-up tables, in
 44   
  * contrast to the Sarwate algorithm [2]. It reads <code>32 bytes</code> at a
 45   
  * time.
 46   
  * <p>
 47   
  * For a detailed description of the Slicing-by-x algorithms look at [1].
 48   
  * <h3>Lookup Table Generation</h3> A lookup table is easy to create, but needs
 49   
  * some knowhow of the internals. So this sections should teach you the internal
 50   
  * knowhow needed for this process.
 51   
  * <p>
 52   
  * There are <code>256</code> different possibilities with <code>8</code> bits.
 53   
  * So a table needs <code>256</code> entries. Each entry holds the remainder of
 54   
  * the polynom division (modulo 2) and the given generator polynom. Thus, an
 55   
  * entry has to be <code>32</code> bits wide, because the remainder has an
 56   
  * degree smaller or equal the degree of the generator polynom.
 57   
  * <p>
 58   
  * There must be tables for the offsets <code>32</code>, <code>40</code>,
 59   
  * <code>48</code>, and <code>56</code>. The <code>256</code> possibilities must
 60   
  * be left shifted by the offset and divided by the generator polynom. These
 61   
  * remainders are stored in the tables.
 62   
  * <p>
 63   
  * <b>Caution:</b> The bits has to be interpreted in reverse order. So the input
 64   
  * bits <code>1 1 0 0 1</code> become to <code>1 0 0 1 1</code>.
 65   
  * <p>
 66   
  * For an implementation see at the class <code>CRCTables</code>.
 67   
  * <p>
 68   
  * An detailed description of this generation process and many more informations
 69   
  * can be found in [3].
 70   
  * <p>
 71   
  * <h3>References</h3>
 72   
  * <p>
 73   
  * [1] Michael E. Kounavis and Frank L. Berry: 'A Systematic Approach to
 74   
  * Building High Performance, Software-based, CRC Generators', Intel Research
 75   
  * and Development, 2005.
 76   
  * <p>
 77   
  * [2] Dilip V. Sarwate: 'Computation of Cyclic Redundancy Checks via Table
 78   
  * Lookup', Communications of the ACM, 1988.
 79   
  * <p>
 80   
  * [3] Ross N. Williams: 'A Painless Guide to CRC Error Detection Algorithms',
 81   
  * http://www.ross.net/crc/download/crc_v3.txt, 1993
 82   
  *
 83   
  * @author Volker Wildi
 84   
  * @see org.jscsi.digest.CRCTables
 85   
  */
