 /**
 33   
  * <h1>CRC32CDigest</h1>
 34   
  * <p>
 35   
  * This class implements a Cyclic Redundancy Code (CRC) with the following generator polynom
 36   
  * <code>0x11EDC6F41</code>. <code>G(x) = x**32 + x**28 + x**27 + x**26 + x**25 + x**23 + x**22 + </code>
 37   
  * <code>x**20 + x**19 + x**18 + x**14 + x**13 + x**11 + x**10 + x**9 + </code> <code>x**8 + x**6 + 1</code>
 38   
  * This polynom has a degree of <code>32</code> and is defined in the iSCSI Standard (RFC3720). Further, this
 39   
  * class implements the algorithm from Intel, which is called Slicing-by-4. This algorithm increases the
 40   
  * performance of the CRC calculation by a factor of <code>2</code> by using <code>4KB</code> of memory for
 41   
  * the look-up tables, in contrast to the Sarwate algorithm [2]. It reads <code>32 bytes</code> at a time.
 42   
  * <p>
 43   
  * For a detailed description of the Slicing-by-x algorithms look at [1].
 44   
  * <h3>Lookup Table Generation</h3> A lookup table is easy to create, but needs some knowhow of the internals.
 45   
  * So this sections should teach you the internal knowhow needed for this process.
 46   
  * <p>
 47   
  * There are <code>256</code> different possibilities with <code>8</code> bits. So a table needs
 48   
  * <code>256</code> entries. Each entry holds the remainder of the polynom division (modulo 2) and the given
 49   
  * generator polynom. Thus, an entry has to be <code>32</code> bits wide, because the remainder has an degree
 50   
  * smaller or equal the degree of the generator polynom.
 51   
  * <p>
 52   
  * There must be tables for the offsets <code>32</code>, <code>40</code>, <code>48</code>, and <code>56</code>
 53   
  * . The <code>256</code> possibilities must be left shifted by the offset and divided by the generator
 54   
  * polynom. These remainders are stored in the tables.
 55   
  * <p>
 56   
  * <b>Caution:</b> The bits has to be interpreted in reverse order. So the input bits <code>1 1 0 0 1</code>
 57   
  * become to <code>1 0 0 1 1</code>.
 58   
  * <p>
 59   
  * For an implementation see at the class <code>CRCTables</code>.
 60   
  * <p>
 61   
  * An detailed description of this generation process and many more informations can be found in [3].
 62   
  * <p>
 63   
  * <h3>References</h3>
 64   
  * <p>
 65   
  * [1] Michael E. Kounavis and Frank L. Berry: 'A Systematic Approach to Building High Performance,
 66   
  * Software-based, CRC Generators', Intel Research and Development, 2005.
 67   
  * <p>
 68   
  * [2] Dilip V. Sarwate: 'Computation of Cyclic Redundancy Checks via Table Lookup', Communications of the
 69   
  * ACM, 1988.
 70   
  * <p>
 71   
  * [3] Ross N. Williams: 'A Painless Guide to CRC Error Detection Algorithms',
 72   
  * http://www.ross.net/crc/download/crc_v3.txt, 1993
 73   
  *
 74   
  * @author Volker Wildi
 75   
  * @see org.jscsi.digest.CRCTables
 76   
  */
