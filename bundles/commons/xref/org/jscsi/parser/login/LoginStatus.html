32  /**
33   * <h1>LoginStatus</h1>
34   * <p>
35   * This enumeration contains only all defined Login Stati, which can send in a login response message in the
36   * iSCSI Protocol (RFC3720).
37   * <p>
38   * <table border="1">
39   * <tr>
40   * <td>0</td>
41   * <td>Success - indicates that the iSCSI target successfully received, understood, and accepted the request.
42   * The numbering fields ( <code>StatSN</code>, <code>ExpCmdSN</code>, <code>MaxCmdSN</code>) are only valid if
43   * Status-Class is <code>0</code>.</td>
44   * </tr>
45   * <tr>
46   * <td>1</td>
47   * <td>Redirection - indicates that the initiator must take further action to complete the request. This is
48   * usually due to the target moving to a different address. All of the redirection status class responses MUST
49   * return one or more text key parameters of the type &quot;TargetAddress&quot;, which indicates the target's
50   * new address. A redirection response MAY be issued by a target prior or after completing a security
51   * negotiation if a security negotiation is required. A redirection SHOULD be accepted by an initiator even
52   * without having the target complete a security negotiation if any security negotiation is required, and MUST
53   * be accepted by the initiator after the completion of the security negotiation if any security negotiation
54   * is required.</td>
55   * </tr>
56   * <tr>
57   * <td>2</td>
58   * <td>Initiator Error (not a format error) - indicates that the initiator most likely caused the error. This
59   * MAY be due to a request for a resource for which the initiator does not have permission. The request should
60   * not be tried again.</td>
61   * </tr>
62   * <tr>
63   * <td>3</td>
64   * <td>Target Error - indicates that the target sees no errors in the initiatorâ€™s Login Request, but is
65   * currently incapable of fulfilling the request. The initiator may re-try the same Login Request later. The
66   * table below shows all of the currently allocated status codes. The codes are in hexadecimal; the first byte
67   * is the status class and the second byte is the status detail.</td>
68   * </tr>
69   * </table>
70   * 
71   * @author Volker Wildi
72   */
